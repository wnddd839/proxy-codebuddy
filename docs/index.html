Cursor Proxy

自建 OpenAI 兼容网关。把 Cursor Direct 与 CodeBuddy 协议直连收成一套可部署的 `/v1` 服务，附带管理台与账号池。

查看仓库 阅读 README

Capabilities

一套网关，两条上游

面向自有账号授权场景：统一入口、统一鉴权、统一管理，再接到 NewAPI 或任意 OpenAI 兼容客户端。

01

OpenAI 兼容 `/v1`

`GET /v1/models` 与 `POST /v1/chat/completions`，用熟悉的 Bearer Key 接入，无需改客户端协议。

02

Cursor Direct

直连 Cursor 上游，账号池轮询，支持常用模型别名，适合把多个 Cursor 账号收成一个渠道。

03

CodeBuddy 协议直连

默认 `protocol_direct`：OAuth 登录后走协议；模型列表来自 `/v3/config`；管理台展示官网真实 Credits（剩余 / 总额）。

04

管理台与写盘保护

`/direct-admin/` 完成认证、启用禁用、刷新与用量查询。账号池拒绝意外缩减，刷新失败只记错误不清空列表。

Architecture

为自建而生

Node 直跑或 Docker Compose；Nginx 可统一暴露公网端口。默认 CodeBuddy 站点为国内站 codebuddy.cn。

你得到什么

  • 单一 `/v1` 入口，对接 NewAPI / 自有客户端
  • 管理台品牌 CodeBuddy Proxy，OAuth 一键导入
  • systemd / Compose 部署文件开箱可用
  • 密钥与账号文件默认不进仓库

Get started

克隆仓库，十分钟起服务

需要 Node.js 20+。复制 `.env.example`，设置 API Key 与管理密码，然后 `npm start`。
